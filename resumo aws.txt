# ** Fundamentos da Computação em Nuvem**
Computação em núvem é a entrega de recursos de TI (servidores, armazenamentos, rededs, banco de dados, etc) pela internet, sob demanda e com pagamento conforme uso. 

Possui como caracteristicas principais:
 - Elasticidade: Aumenta ou diminui instantaneamente os recursos conforme a necessidade;
 - Escalabilidade: Capacidade de crescer sem grandes invetimentos iniciais;
 - Sob demanda: você paga apenas pelo que usa, sem comprometer o hardware;
 - Alta disponibilidade: Serviços distribuidos para evitar falhas.

Tipos de Computação em nuvem:
 - Infraestrutura as a Service (IaaS): Infraestrutura básica (servidores, redes, armazenamentos). Oferece o mais alto nível de flexibilidade e controle de gerenciamento sobre os recursos de TI.
 - Platform as a Service (PaaS): plataforma para desenvolvimetno sem gerenciar infraestrutura (não há necessidade de se preocupar com aquisição de recursos, planejamento de capacidade, manutenção de software, correção ou qualquer outro tipo de trabalho genérico necessário para execução dos aplicativos);
 - Software as a Service (SaaS): geralmente são os aplicativos finais ao usuário (gmail, office365). Não há necessidade em pensar sobre a manutenção do serviço ou gerenciamento da infraestrutura subjacente. Você apenas precisa se preocupar cem como utilizará o softare.

- Comparação: On-Premise (local) × Cloud
 O modelo tradicional (on-premise/local) possui alto investimento inicial (compra de servidores, licenças, espaços físcos), custos contínuos de manutenção e capacidade fixa (escalabilidade é limitada).
 Em comparação, o Cloud computing ppossui o custo baseado no pay-as-you-go (sob demanda), escalabilidade alta e rápida, e a responsável pela manutenção da infraestrutura é a própria AWS.
 O modelo híbrido combina as duas formas: on-premise e cloud. É útil para empresas que precisam manter parte dos dados localmente (por complience ou legado) e usar nuvem para escalabilidade.

# **Infraestrutura da AWS**
É o conjunto de recursos físicos (data centers, redes, servidores e sistemas de energia) que sustentam todos os serviços de nuvem oferecidos pela plataforma. Possui 38 Regiões geográficas (onde ficam localizadas as data centers [Am. do Norte, Europa, Ásia-Pacífico]), totalizando 120 Zonas de dispoonibilidade (Data centers isolados dentro dedd uma região, para garantir alta disponibilidade) e locais de borda/ edge locations (pontos de presença distribuidos em várias cidades e países, usados para entregar conteúdo de forma mais rápida e reduzir a latência).
Essa infraestrutura foi projetada para entregar maior segurança, disponibilidade, performance, escalabilidade  e flexibilidade, permitindo que empresas de qualquer porte executem aplicações críticas sem manter servidores próprios. 
A AWS permite provisionar recursos sob demanda, sem necessidade de compra antecipada de hardware.  Ele pode ser manual (o cliente define o tamanho e quantidade) ou automático (serviços ajustam os recursos dinamicamente [auto scaling]).

# **Computação em AWS**
- *Amazon EC2 (Elastic Compute Cloud)*: Permite xriar e gerenciar instâncias de servidores virtuais na nuvem. É possível escolher o sistema operacional, o CPU, memória, armazenamento e tipo de rede conforme a necessidade. Ele oferece controle total sobre o aambiente, semelhante a um servidor físico tradicional, porém com a vantagem da elsaticidade da nuvem. Ele suporta diversos tipos de instâncias, otimizadas para computação, memória, armazenamento ou workloads gráficos, além de diferentes modelos como, on- Demand, reserved instances e Spot Instances.

- *Amazon EC2 Auto Scaling*: Ajusta automaticamente o número de isntancias EC2 conforme a demanda. Ele permite o monitoramento das instências em execução, verificação de integridade personalizadas, Equilibrio entre Zonas de disponibiliade, Varios tipos de instâncias e várias opções de compra, substituição automatizada de instâncias spot, balancemaewnto de carga, escalabilidade, atualização de instância, Hooks do ciclo de vida e suporte para cargas de trabalho com estado.

- *Elastic Load Balancing*: O Elastic Load Balancing distribui automaticamente o tráfego de entrada entre múltiplos recursos de computação, como instâncias EC2, containers ou endereços IP. Ele melhora a alta disponibilidade, a tolerância a falhas e o desempenho das aplicações. A AWS oferece diferentes tipos de load balancers, adequados para aplicações HTTP/HTTPS, TCP/UDP e arquiteturas modernas baseadas em microsserviços.

- *Serviços de mensageria*: Os serviços de mensageria da AWS permitem a comunicação assíncrona entre componentes de sistemas distribuídos, desacoplando produtores e consumidores de mensagens. Eles são usados para processamento assíncrono, integração entre sistemas, orquestração de eventos, aumetno de resiliencia e escalabilidade. 

- *Computação sem servidor (Serverless)*:No modelo serverless, o desenvolvedor não precisa se preocupar com servidores, sistema operacional ou escalabilidade. A AWS gerencia automaticamente a infraestrutura, enquanto o usuário paga apenas pelo tempo de execução do código. Esse modelo é ideal para:

Processamento sob demanda

APIs

Automação

Aplicações orientadas a eventos

## **Ferramentas possiveis para o projeto** 
- **Amazon EC2**: Para hospedar servidores virtuais que vão rodar o sistema principal da farmácia. Isso permite flexibilidade para ajustar recursos conforme a demanda.

- **Amazon RDS**: Para o banco de dados da farmácia, caso ele seja relacional (como MySQL ou PostgreSQL). O RDS cuida de backups, atualizações e manutenção, deixando tudo mais fácil.

- **Amazon S3: Para armazenar arquivos, relatórios, imagens de produtos e qualquer outro dado que precise de armazenamento seguro e escalável.

- **AWS Lambda: Para automatizar pequenas tarefas, como processar atualizações de estoque ou enviar notificações, sem precisar manter servidores dedicados.

- **Amazon VPC: Para criar uma rede privada virtual, garantindo que o sistema da farmácia tenha um ambiente de rede isolado e seguro.

- **AWS IAM: Para gerenciar quem pode acessar o quê, criando usuários e definindo permissões, mantendo a segurança dos dados.

- **Amazon CloudWatch: Para monitorar o desempenho do sistema, recebendo alertas se algo precisar de atenção, e garantindo que tudo funcione sem interrupções.

- **AWS Amplify**: O AWS Amplify é uma ferramenta muito útil se você precisar criar ou migrar aplicações web ou móveis. Ele simplifica o desenvolvimento front-end e a conexão com serviços de back-end na AWS. Basicamente, o Amplify ajuda a construir interfaces de usuário, autenticação de usuários, armazenamento de dados e até mesmo hospedagem de sites estáticos de forma bem integrada.

- **AWS Cost Explorer**: O AWS Cost Explorer é uma ferramenta que ajuda você a entender e gerenciar os custos na AWS. Com ele, você pode visualizar seus gastos ao longo do tempo, identificar padrões de uso e descobrir onde é possível otimizar para economizar. Isso é especialmente útil para garantir que a migração do sistema da farmácia fique dentro do orçamento.

- **AWS Shield e WAF**: Além dessas, você pode considerar o AWS Shield, que é um serviço de proteção contra ataques DDoS, e o AWS WAF (Web Application Firewall), que ajuda a proteger suas aplicações web contra exploits e acessos indesejados. Essas ferramentas são importantes para garantir a segurança do sistema.

- **Amazon Pinpoint**: Essa ferramenta é excelente para engajamento de clientes. Você pode usar o Pinpoint para campanhas de marketing direcionadas, enviar promoções por e-mail, SMS ou notificações push para os clientes da farmácia, ajudando a aumentar o relacionamento e as vendas.

- **Amazon Connect**: Se a farmácia precisar de um sistema de atendimento ao cliente por voz ou chat, o Amazon Connect é uma solução de contact center em nuvem que pode ser facilmente implementada. Isso pode melhorar bastante o suporte ao cliente.

- **AWS AppSync**: Se a farmácia tiver um aplicativo móvel ou um sistema que precise sincronizar dados em tempo real (como inventário ou informações de produtos), o AppSync facilita a criação de APIs flexíveis para isso.

- **AWS IoT Core**: Se a farmácia estiver interessada em dispositivos conectados, como sensores de temperatura para medicamentos sensíveis, o IoT Core permite integrar esses dispositivos com facilidade.
